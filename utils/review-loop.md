# Review Loop

レビュー → 修正 → 再レビューを1サイクルとして繰り返す。
`review.md` と `fix-from-review.md` を組み合わせたワンストップフロー。

- **目的:** レビュー指摘がゼロになるまで修正サイクルを回す
- **入力:** レビュー対象ブランチ・ベースブランチ・最大ループ回数（デフォルト3）
- **出力:** 修正済みコード + 各イテレーションのレビューレポート

---

## 事前確認事項

1. **ブランチ名** — レビュー対象のブランチ
2. **ベースブランチ** — 差分の比較元（例: `main`）
3. **最大ループ回数** — 無限ループ防止（デフォルト: 3回）
4. **ドキュメント更新の有無** — `create-docs.md` の生成物がある場合は `fix-from-review.md` のフローも適用する

---

## 実行フロー

### ループ開始前

```bash
# 差分を確認
git diff {ベースブランチ}...HEAD
```

---

### 各イテレーション（N回目）

#### Step 1: レビュー実行

`review.md` の手順に従い、`{ブランチ名}.review-N.md` を生成する。
（N はイテレーション番号。初回は `review-1.md`）

#### Step 2: 指摘の確認

レビューレポートの改善提案（Suggestions）を確認する。

- **指摘が0件** → ループ終了（Phase: 完了報告へ）
- **指摘がある** → Step 3 へ

#### Step 3: 修正の実施

指摘に対して以下のいずれかを実施:

**ドキュメントがある場合:**
`fix-from-review.md` の手順に従い、requirement.md / spec.md / task.md を更新する。

**ドキュメントがない場合（コードへの直接修正）:**
各指摘について:
- 指摘内容と提案コードを確認
- コードを修正
- 修正内容をユーザーに提示・確認

#### Step 4: 修正の確定

```bash
git add -A
git commit -m "fix: レビュー指摘を修正（イテレーション N）"
```

→ 次のイテレーションへ（Step 1 に戻る）

---

### 完了報告

- 実施したイテレーション数
- 各イテレーションの指摘件数の推移（例: 1回目 5件 → 2回目 1件 → 3回目 0件）
- 最終レビューレポートのパス
- 修正したファイルの一覧

### 最大ループ到達時

上限に達しても指摘が残っている場合:

- 残存する指摘の一覧を提示
- 継続するか・残りを手動対応するかをユーザーに確認
