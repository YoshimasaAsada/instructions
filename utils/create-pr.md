# Create PR Description

差分とコミット履歴からPR説明文を生成する。

- **目的:** 変更内容・影響範囲・テスト手順を含むPR説明文を作成する
- **入力:** ベースブランチとの差分・コミット履歴・PRテンプレート（あれば）
- **出力:** PR説明文（Markdown）

---

## 事前確認事項

以下が未確定の場合はユーザーに確認する:

1. **ベースブランチ** — マージ先（例: `main`, `develop`）
2. **PRテンプレート** — リポジトリにテンプレートがあればそれに従う

---

## 実行フロー

### Phase 1: 情報収集

以下を並列で取得する:

```bash
# コミット一覧
git log --oneline {ベースブランチ}..HEAD

# 差分の統計
git diff --stat {ベースブランチ}...HEAD

# 差分の全文
git diff {ベースブランチ}...HEAD
```

PRテンプレート（`.github/PULL_REQUEST_TEMPLATE.md` など）が存在すればそちらも読み込む。

---

### Phase 2: 変更の分析

差分を読み、以下を把握する:

- **変更の種別**: 新機能 / バグ修正 / リファクタリング / ドキュメント / テスト
- **影響範囲**: 変更されたレイヤー（DB / バックエンド / フロントエンド / インフラ）
- **破壊的変更の有無**: API変更・DB変更・環境変数追加など
- **関連Issue/チケット**: コミットメッセージやブランチ名から推定

---

### Phase 3: PR説明文の生成

PRテンプレートがある場合はそれに従う。ない場合は以下の形式:

```markdown
## 概要

（変更の目的と背景を1〜3文で）

## 変更内容

- （変更点を箇条書き。ファイル・関数名を含める）
- ...

## 影響範囲

- （影響を受けるレイヤー・機能を列挙）

## 破壊的変更

（ある場合のみ記載。API変更・DB変更・環境変数追加など）

## テスト手順

1. （手動確認が必要な場合の手順）
2. ...

## 関連Issue

- #{Issue番号}
```

---

### Phase 4: 出力

- 生成したPR説明文をユーザーに提示する
- `gh pr create` コマンドを提示する（実行はユーザーに確認してから）
