# Create Onboarding Guide

コードベースを分析し、新しくプロジェクトに参加するメンバー向けのオンボーディングガイドを生成する。

- **目的:** プロジェクトの全体像・セットアップ手順・開発フローを新メンバーが理解できるガイドを作成する
- **入力:** 対象リポジトリ + 読者の想定（バックエンド / フロントエンド / フルスタック等）
- **出力:** オンボーディングガイド（Markdown）

---

## 事前確認事項

1. **対象リポジトリ** — ガイドを作るリポジトリ
2. **読者の想定** — バックエンドエンジニア / フロントエンドエンジニア / フルスタック / 非エンジニアなど
3. **重点的に説明すべき領域** — 特にあればユーザーに確認

---

## 実行フロー

### Phase 1: プロジェクトの全体調査

以下を調査する:

- **package.json / go.mod / requirements.txt** 等 — 使用技術・依存ライブラリ
- **ディレクトリ構成** — アーキテクチャの概観
- **README.md** — 既存のドキュメント
- **設定ファイル** — `.env.example`, `docker-compose.yml`, CI設定など
- **DB定義** — ORMスキーマ / マイグレーション
- **エントリポイント** — `main.ts`, `index.ts`, `app.py` など
- **テスト** — テストの構成と実行方法

---

### Phase 2: ガイドの構成

以下のセクションで構成する:

```markdown
# {プロジェクト名} オンボーディングガイド

## プロジェクト概要
（何をするプロジェクトか。ビジネス上の目的。1-3文で）

## 技術スタック
| カテゴリ | 技術 |
|---------|------|
| 言語 | ... |
| フレームワーク | ... |
| DB | ... |
| インフラ | ... |
| CI/CD | ... |

## アーキテクチャ概要
（レイヤー構成・主要なデータフローの説明）

## ディレクトリ構成
（主要ディレクトリの役割。深さ2階層程度）

## ローカル開発環境のセットアップ
1. 前提条件（Node.js, Docker 等のバージョン）
2. リポジトリのクローン
3. 依存関係のインストール
4. 環境変数の設定
5. DBのセットアップ
6. 開発サーバーの起動
7. 動作確認

## 主要な概念・ドメイン用語
| 用語 | 説明 |
|------|------|
| ... | ... |

## 開発フロー
- ブランチ戦略
- コミットルール
- PR作成〜マージの流れ
- テストの実行方法
- デプロイの流れ

## よくある開発タスクの例
（「APIエンドポイントを追加するには」「テストを追加するには」等、具体的な手順）

## 困ったときに見る場所
- ログの確認方法
- エラーの調べ方
- 質問先（Slack チャンネル、ドキュメント等）
```

---

### Phase 3: 生成と出力

- ガイドをMarkdownファイルとして出力
- コードベースから自動検出できなかった項目には `<!-- TODO: 要記入 -->` マーカーを残す
- 出力先のパスとTODO箇所の数を報告
