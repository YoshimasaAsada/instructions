# Claude Code 運用ルール

このリポジトリはAIへの指示書を一元管理するためのものです。

## 「コミットして」と言われたときの手順

以下を**必ず順番通りに**実行してください。

### 1. 変更内容の把握

```bash
git status
git diff HEAD
```

### 2. README.md の Index を更新する

`README.md` の `<!-- INDEX_START -->` から `<!-- INDEX_END -->` の間を、
リポジトリ内の全 `.md` ファイル（README.md と CLAUDE.md を除く）を走査して再生成する。

各エントリの説明は、対象ファイルの内容を読んで**1行で要約**すること。

ディレクトリごとにセクションを分けて出力する:

```
### utils/（汎用）

| ファイル | 説明 |
|----------|------|
| [utils/foo.md](utils/foo.md) | 説明 |

### claude/（Claude Code 固有）

| ファイル | 説明 |
|----------|------|
| [claude/bar.md](claude/bar.md) | 説明 |
```

ファイルが存在しないディレクトリのセクションは出力しない。

### 3. コミットメッセージを生成する

- `git diff HEAD` の内容を読み、変更の意図・種別・影響範囲を把握する
- 日本語でコミットメッセージを作成する
- フォーマット:
  ```
  <種別>: <変更内容の要約>

  <詳細（必要な場合のみ）>
  ```
- 種別の例: `add`（追加）、`update`（更新）、`fix`（修正）、`remove`（削除）、`refactor`（整理）

### 4. ステージングとコミット

- README.md の更新を含む変更ファイルをステージングする
- 生成したメッセージでコミットを作成する

## ディレクトリ命名規則

| ディレクトリ | 用途 |
|------------|------|
| `utils/`   | 汎用指示書（どのAIツールでも利用可能なもの） |
| `claude/`  | Claude Code 固有の指示書（Explore エージェント等 Claude の機能に依存するもの） |
| `copilot/` | GitHub Copilot 固有の指示書 |
| `domains/` | ドメイン・業務知識系の指示書 |

新しいカテゴリが必要な場合は上記テーブルにも追記すること。

### 配置の判断基準

- Claude Code の機能（Explore エージェント、ツール呼び出しなど）に言及している → `claude/`
- GitHub Copilot の機能（`@workspace`、`/` コマンドなど）に言及している → `copilot/`
- 特定ツールに依存せず、どのAIにそのまま貼り付けて使える → `utils/`

## 指示書ファイルの書き方

- 先頭に `# Instruction` または役割名の見出しを置く
- 出力形式が決まっている場合はテンプレートを末尾に記載する
- 言語は日本語・英語どちらでも可
